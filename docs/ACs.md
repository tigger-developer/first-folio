## Issue 9: Manuscript PDF Mode

GitHub issue: https://github.com/tadg-paul/first-folio/issues/9

| ID | AC | Tests |
|---|---|---|
| AC9.1 | Given a manuscript command with Markdown or org-mode chapter files and a PDF target, First Folio creates one manuscript PDF at the target path. | ✅ RT-9.1: Multiple Markdown chapter fixtures create a non-empty PDF target through the manuscript CLI<br>✅ RT-9.2: Multiple org-mode chapter fixtures create a non-empty PDF target through the manuscript CLI<br>✅ RT-9.3: A `.typ` target creates Typst source instead of compiling PDF<br>✅ UT-9.1: A human reviewer renders a real chapter range and confirms the resulting PDF opens in a PDF viewer |
| AC9.2 | Given a quoted chapter glob or multiple explicit chapter paths, First Folio includes only the selected chapters in deterministic chapter order with a blank-line separator between files. | ✅ RT-9.4: An unsorted explicit path list is rendered in deterministic chapter order<br>✅ RT-9.5: A quoted glob matching a subset excludes non-matching chapters<br>✅ RT-9.6: A glob with no matches exits non-zero with a clear error<br>✅ RT-9.7: Adjacent Markdown files without trailing newlines do not merge prose and the next heading<br>✅ RT-9.8: Adjacent org-mode files without trailing newlines do not merge prose and the next heading |
| AC9.3 | Given manuscript Markdown input, prose paragraphs, headings, parts, chapters, scene breaks, emphasis, private notes, and footnotes have corresponding manuscript events or Typst markup. | ✅ RT-9.9: Paragraph fixtures appear as manuscript body text in generated Typst<br>✅ RT-9.10: Part and chapter heading fixtures produce manuscript part and chapter pages in generated Typst<br>✅ RT-9.11: Emphasis, scene break, and footnote fixtures appear as corresponding Typst markup<br>✅ RT-9.12: Private-note fixtures are excluded from manuscript output<br>✅ UT-9.2: A human reviewer checks a rendered Markdown sample for acceptable manuscript typography |
| AC9.4 | Given manuscript org-mode input, prose paragraphs, headings, parts, chapters, scene breaks, emphasis, noexport sections, and footnotes have corresponding manuscript events or Typst markup. | ✅ RT-9.13: Org paragraph fixtures appear as manuscript body text in generated Typst<br>✅ RT-9.14: Org part and chapter heading fixtures produce manuscript part and chapter pages in generated Typst<br>✅ RT-9.15: Org emphasis, scene break, and footnote fixtures appear as corresponding Typst markup<br>✅ RT-9.16: Org `:noexport:` fixtures are excluded from manuscript output<br>✅ UT-9.3: A human reviewer checks a rendered org-mode sample for acceptable manuscript typography |
| AC9.5 | Given existing root `folio.font`, `folio.font-size`, `folio.page`, and `folio.margin` configuration, script, letter, and manuscript rendering inherit the same document defaults unless a mode or child override is present. | ✅ RT-9.17: Root `folio.page: a4` appears in generated Typst for script, letter, and manuscript outputs<br>✅ RT-9.18: Root `folio.font` appears in generated Typst for script, letter, and manuscript body text<br>✅ RT-9.19: Root `folio.font-size` appears in generated Typst for script, letter, and manuscript body text<br>✅ RT-9.20: Existing `folio.letter.font` and `folio.letter.font-size` override root defaults for cover letters only |

| AC9.7 | Given manuscript-specific font configuration, manuscript child font settings override the manuscript mode defaults for only their own elements. | ✅ RT-9.25: `folio.manuscript.font` overrides root `folio.font` for manuscript body text<br>✅ RT-9.26: `folio.manuscript.heading-font` overrides root `folio.heading-font` for manuscript headings<br>✅ RT-9.27: `folio.manuscript.mono-font` applies to inline code and fenced code blocks only<br>✅ RT-9.28: `folio.manuscript.title-font`, `author-font`, `date-font`, and `version-font` apply only to the corresponding title-page elements<br>✅ RT-9.29: `folio.manuscript.page-header.font` applies only to the running page header |
| AC9.8 | Given British manuscript defaults, First Folio uses a publisher-friendly A4 manuscript layout with 20mm margins, a title page, running page header, new-page chapters, and part divider pages. | ✅ RT-9.30: British manuscript preset uses A4, 20mm margins on all sides, Libertinus Serif body text, and root-compatible heading defaults<br>✅ RT-9.31: A running header renders as author full name, book title, and page number at the top right, 20mm from the page edge<br>✅ RT-9.32: Body/title content begins below the running header with configured top padding<br>✅ RT-9.33: Each chapter starts on a new page about one third down the page<br>✅ RT-9.34: Each part starts on a separate page centred vertically and horizontally<br>✅ UT-9.4: A human reviewer checks a British manuscript sample for acceptable publisher-facing layout<br>✅ UT-9.6: A human reviewer checks the dummy Markdown manuscript rendered with the British preset<br>✅ UT-9.7: A human reviewer checks the dummy org-mode manuscript rendered with the British preset |
| AC9.9 | Given US manuscript defaults, First Folio applies a Shunn-based manuscript override YAML on top of the British manuscript base without making US layout the global default. | ✅ RT-9.35: `us-overrides-manuscript.yaml` deep-merges on top of `british-manuscript.yaml` and changes only specified manuscript keys<br>✅ RT-9.36: US manuscript override keeps page size inherited unless `folio.manuscript.page` is explicitly configured<br>✅ RT-9.37: US manuscript override uses a monospaced manuscript font family for body text<br>✅ RT-9.38: US manuscript title page contains title, subtitle, author attribution, date, word count, address, email, website, and version when those fields are configured<br>✅ RT-9.39: US manuscript text pages use the shared running header format and chapter placement contract<br>✅ RT-9.40: US manuscript override uses `line-spacing: 2`, `paragraph-indent: 12.7mm`, and `paragraph-spacing: 0`<br>✅ UT-9.5: A human reviewer checks a US manuscript sample against the Shunn-based layout<br>✅ UT-9.8: A human reviewer checks the dummy Markdown manuscript rendered with the US preset<br>✅ UT-9.9: A human reviewer checks the dummy org-mode manuscript rendered with the US preset |
| AC9.10 | Given no page configuration, every First Folio rendering mode uses A4 as the page size. | ✅ RT-9.41: Script output uses A4 with no page config<br>✅ RT-9.42: Letter output uses A4 with no page config<br>✅ RT-9.43: Manuscript output uses A4 with no page config<br>✅ RT-9.44: Selecting an American, screenplay, or US manuscript style does not change page size unless a page value is explicitly configured |

| AC9.12 | Given manuscript TOC configuration, First Folio includes a table of contents by default and applies TOC-specific font and layout overrides when present. | ✅ RT-9.50: Manuscript output includes a table of contents when no TOC config is present<br>✅ RT-9.51: `folio.manuscript.toc.enabled: false` excludes the table of contents<br>✅ RT-9.52: `folio.manuscript.toc.font`, `font-size`, and `font-weight` affect TOC entries only<br>✅ RT-9.53: `folio.manuscript.toc.heading-font`, `heading-font-size`, and `heading-font-weight` affect the TOC heading only<br>✅ RT-9.54: TOC entries include part and chapter headings in document order |
